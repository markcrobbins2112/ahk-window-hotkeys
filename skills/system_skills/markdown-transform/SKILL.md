# AIMD Markdown Transformation Skill
<a id="a-aimdmarkdowntransformationskill"></a>[TOC](#toc-aimdmarkdowntransformationskill)

This skill provides comprehensive instructions, algorithms, and guidelines to programmatically or manually convert unstructured, loose, or tool-specific Markdown workspaces (such as those generated by desktop tools like Obsidian using proprietary formats like Wikilinks) into strict, standard-compliant **GitHub Flavored Markdown (GFM)** documentation.

---

## 🔍 1. Core Conversion Rulebook & Strategies
<a id="a-1coreconversionrulebookstrategies"></a>[TOC](#toc-1coreconversionrulebookstrategies)

### Rule 1.1: Wikilink Conversion & Relinking
<a id="a-rule11wikilinkconversionrelinking"></a>[TOC](#toc-rule11wikilinkconversionrelinking)
Wikilinks (`[[FileName]]` or `[[FileName|Custom Label]]`) are proprietary. They must be extracted and replaced with robust, standard GFM relative links.

- **Extraction Pattern**: 
  - Match wikilinks using the Regular Expression:
    ```regex
    /\[\[([^\]|]+)(?:\|([^\]]+))?\]\]/g
    ```
- **Conversion Strategy**:
  - **Internal files**: Convert `[[About Us]]` to `[About Us](About%20Us.md)` (or standard relative file path).
  - **With labels**: Convert `[[[AIMD/CODE.md](../../../AIMD/CODE.md)|Code Guidelines]]` to `[Code Guidelines](AIMD/CODE.md)`.
  - **Internal header targets**: If linking to a local header, use slug-anchored relative hashes, e.g., `[[#🎯 Primary Goals]]` becomes `[Primary Goals](#a-primarygoals)`.

### Rule 1.2: Raw HTML Anchor Injection & TOC Binding
<a id="a-rule12rawhtmlanchorinjectiontocbinding"></a>[TOC](#toc-rule12rawhtmlanchorinjectiontocbinding)
Every heading (H1, H2, H3, H4, H5, H6) must be mapped to a clean, unique anchor ID that guarantees native jump-target routing, combined with a quick return-to-top Table of Contents (`[TOC]`) link.

- **Injection Format**:
  Immediately below the header line, append a raw HTML anchor block and a TOC link:
  ```markdown
  ### 🎯 Primary Goals
  <a id="a-primarygoals"></a>[TOC](#toc-primarygoals)
  ```
- **ID Prefixing Standard**:
  - Target anchor IDs **MUST** start with the `a-` prefix.
  - The TOC return-link hash **MUST** match with the corresponding `toc-` prefix (e.g., `[TOC](#toc-primarygoals)`).

### Rule 1.3: Heading Serialization (Preventing MD024 Linter Collisions)
<a id="a-rule13headingserializationpreventingmd024lintercollisions"></a>[TOC](#toc-rule13headingserializationpreventingmd024lintercollisions)
Duplicate heading content is strictly prohibited by standard linters (MD024). To prevent collisions, duplicate headings must be serialized:

- **Incremental Suffixes**:
  If a heading occurs multiple times in a document, append an incremental index suffix starting with `-1` (e.g., `<a id="a-settings"></a>`, `<a id="a-settings-1"></a>`, `<a id="a-settings-2"></a>`).
- **Timestamp Suffixes (Log Entries)**:
  For subheadings nested under unique date/time records (e.g., `🎯 Primary Goals & Requirements` occurring under logs like `### 📅 [2026-07-22]`), append the parent's sanitized timestamp directly to the ID to ensure absolute uniqueness (e.g., `<a id="a-primarygoalsrequirements20260722"></a>`).

### Rule 1.4: Workspace File Placement & Naming
<a id="a-rule14workspacefileplacementnaming"></a>[TOC](#toc-rule14workspacefileplacementnaming)
- **Root Directory Scope**: The workspace root (`/`) MUST ONLY contain [`README.md`](../../../README.md) and [`AGENTS.md`](../../../AGENTS.md).
- **AIMD Directory Scope**: All other Markdown documentation items ([`LOG.md`](../../../AIMD/LOG.md), [`TASKS.md`](../../../AIMD/TASKS.md), [`BUILD.md`](../../../AIMD/BUILD.md), [`CODE.md`](../../../AIMD/CODE.md), [`FEATURES.md`](../../../AIMD/FEATURES.md), [`MANUAL.md`](../../../AIMD/MANUAL.md), [`SPEC.md`](../../../AIMD/SPEC.md), [`TESTING.md`](../../../AIMD/TESTING.md), `+ce-qp.md`, etc.) are placed inside `./AIMD/`.
- **Naming Conventions**:
  - The log document is named [`LOG.md`](../../../AIMD/LOG.md) (uppercase).
  - The task tracking document is named [`TASKS.md`](../../../AIMD/TASKS.md) (uppercase).
- **Exclusion Rule**: [`README.md`](../../../README.md) and [`AGENTS.md`](../../../AGENTS.md) MUST NOT exist inside `./AIMD/`.

## 🛠️ 2. Strict Linter Compliance Checklist
<a id="a-2strictlintercompliancechecklist"></a>[TOC](#toc-2strictlintercompliancechecklist)

To pass strict markdown lint rules (such as `markdownlint`), your transformed file must adhere to these structural laws:

- **Standard Heading Increments (MD001 & MD002)**: Headings must start with a level 1 title (`# TITLE`) and increment sequentially by exactly one level at a time (e.g., never jump directly from `#` to `###`).
- **No Bare URL Links (MD034)**: Never print raw URLs directly in paragraph text. Always wrap raw URLs in angle brackets (e.g., `<https://example.com>`) or form standard GFM links.
- **Spaces Surrounding List Items (MD030)**: Ensure there is exactly one space after a list bullet marker (`-` or `*`) or order number (e.g. use `- Checked item` instead of `-  Checked item`).
- **No Trailing Spaces (MD009)**: Eliminate trailing whitespaces at the end of lines, unless explicitly declaring a hard line break with exactly 2 trailing spaces.
- **Horizontal Rule Consistency (MD035)**: Always use a consistent character and spacing format for horizontal dividers (e.g., `---`).
- **No Hash Symbols in Comments**: This rule applies strictly to the first multi-line HTML comment block (file header comment) and single-line comments that start with `TEMPLATE:` (e.g. `<!-- TEMPLATE: ... -->`). Strip or omit all `#` characters inside these specific comments.

## 🛠️ 5. Master JavaScript Reference Compiler
<a id="a-5masterjavascriptreferencecompiler"></a>[TOC](#toc-5masterjavascriptreferencecompiler)

Below is the definitive, production-ready, zero-dependency Node.js reference implementation of the GFM Markdown standardizer and TOC generator used to compile the workspace docs:

```javascript
import fs from 'fs';
import path from 'path';

/**
 * Standardizes a single Markdown document:
 * 1. Generates URL-friendly, lowercase, alphanumeric-only anchor slugs.
 * 2. Serializes duplicate heading slugs (with unique indexes or timestamp prefixes).
 * 3. Injects clean raw HTML anchor tags and TOC return-to-top links.
 * 4. Compiles and updates a nested, beautifully spaced Table of Contents block.
 *
 * @param {string} filePath - Absolute or relative path to the Markdown file.
 */
function processMarkdownFile(filePath) {
	let content = fs.readFileSync(filePath, 'utf8');
	let lines = content.replace(/\r\n/g, '\n').split('\n');

	let inCodeBlock = false;
	let inCommentBlock = false;
	const headings = [];

	// First pass: locate headings
	for (let i = 0; i < lines.length; i++) {
		const line = lines[i];
		const trimmed = line.trim();

		if (trimmed.startsWith('```')) {
			inCodeBlock = !inCodeBlock;
			continue;
		}
		if (inCodeBlock) continue;

		if (!inCommentBlock && trimmed.startsWith('<!--')) {
			if (!trimmed.endsWith('-->') || trimmed === '<!--') {
				inCommentBlock = true;
			}
			continue;
		}
		if (inCommentBlock) {
			if (trimmed.endsWith('-->') || trimmed.includes('-->')) {
				inCommentBlock = false;
			}
			continue;
		}

		const headingMatch = line.match(/^(#{1,6})\s+(.+)$/);
		if (headingMatch) {
			const level = headingMatch[1].length;
			const text = headingMatch[2].trim();
			
			if (text.toLowerCase().includes('table of contents')) {
				continue;
			}

			headings.push({
				lineIndex: i,
				level,
				text,
				originalLine: line
			});
		}
	}

	if (headings.length === 0) return;

	// Assign slugs and handle serialization/duplicates
	let currentParentTimestamp = null;
	const slugCounts = {};

	for (const heading of headings) {
		const text = heading.text;

		// Extract timestamp from parent date headers e.g. ### [2026-06-01T03:41:00Z]
		const dateMatch = text.match(/\[(\d{4}-\d{2}-\d{2}t\d{2}:\d{2}:\d{2}z)\]/i) || text.match(/\[(\d{4}-\d{2}-\d{2})\]/i);
		if (dateMatch) {
			currentParentTimestamp = dateMatch[1].toLowerCase().replace(/[^a-z0-9]/g, '');
		}

		// Generate base slug
		const baseSlug = text.replace(/<[^>]+>/g, '').toLowerCase().replace(/[^a-z0-9]/g, '');

		// Handle duplicate headers and log entry subheadings
		const isLogSubheading = (
			text.includes('Goals & Requirements') ||
			text.includes('Primary Goals') ||
			text.includes('Completed Changes') ||
			text.includes('Affected Files')
		);

		let slug = baseSlug;
		if (isLogSubheading && currentParentTimestamp) {
			slug = baseSlug + currentParentTimestamp;
		} else {
			if (slugCounts[slug] === undefined) {
				slugCounts[slug] = 0;
			} else {
				slugCounts[slug]++;
				slug = `${slug}-${slugCounts[slug]}`;
			}
		}

		heading.slug = slug;
	}

	// Second pass: insert/update anchors right below headings
	const linesWithAnchors = [];
	const headingMap = new Map();
	for (const h of headings) {
		headingMap.set(h.lineIndex, h);
	}

	let skipLinesCount = 0;
	for (let i = 0; i < lines.length; i++) {
		if (skipLinesCount > 0) {
			skipLinesCount--;
			continue;
		}

		const line = lines[i];
		linesWithAnchors.push(line);

		if (headingMap.has(i)) {
			const h = headingMap.get(i);
			const anchor = `<a id="a-${h.slug}"></a>[TOC](#toc-${h.slug})`;

			// Check for pre-existing anchor lines to skip and replace
			let nextIdx = i + 1;
			while (nextIdx < lines.length && lines[nextIdx].trim() === '') {
				nextIdx++;
			}
			if (nextIdx < lines.length) {
				const nextLineTrimmed = lines[nextIdx].trim();
				if (nextLineTrimmed.startsWith('<a id="a-') && nextLineTrimmed.includes('[TOC](#toc-')) {
					skipLinesCount = nextIdx - i;
				}
			}

			linesWithAnchors.push(anchor);
		}
	}

	// Third pass: find or insert/replace the Table of Contents block
	let tocStart = -1;
	for (let i = 0; i < linesWithAnchors.length; i++) {
		if (linesWithAnchors[i].trim() === '<!-- TOC location -->') {
			tocStart = i;
			break;
		}
	}

	if (tocStart === -1) {
		let firstH1Idx = -1;
		for (let i = 0; i < linesWithAnchors.length; i++) {
			const line = linesWithAnchors[i];
			if (line.startsWith('# ') && !line.toLowerCase().includes('table of contents')) {
				firstH1Idx = i;
				break;
			}
		}

		if (firstH1Idx !== -1) {
			let insertIdx = firstH1Idx + 1;
			if (insertIdx < linesWithAnchors.length && linesWithAnchors[insertIdx].startsWith('<a id="a-')) {
				insertIdx++;
			}
			linesWithAnchors.splice(insertIdx, 0, '', '<!-- TOC location -->', '---');
			tocStart = insertIdx + 1;
		}
	}

	if (tocStart !== -1) {
		let nextHeadingIdx = -1;
		for (let i = tocStart + 1; i < linesWithAnchors.length; i++) {
			const line = linesWithAnchors[i];
			const trimmed = line.trim();
			if (trimmed.startsWith('#') && !trimmed.toLowerCase().includes('table of contents')) {
				nextHeadingIdx = i;
				break;
			}
		}

		if (nextHeadingIdx !== -1) {
			const tocBlock = [];
			tocBlock.push('<!-- TOC location -->');
			tocBlock.push('## 🔍 Table of Contents');
			tocBlock.push('<!-- Maintained by script -->');

			const minLevel = headings.length > 0 ? Math.min(...headings.map(h => h.level)) : 2;

			for (const h of headings) {
				const indent = ' '.repeat(Math.max(0, h.level - minLevel) * 2);
				const itemText = h.text.replace(/<[^>]+>/g, '').trim();
				tocBlock.push(`${indent}- [${itemText}](#a-${h.slug}) <a id="toc-${h.slug}"></a> ^toc-${h.slug}`);
			}
			tocBlock.push('---');

			lines_with_anchors.splice(tocStart, nextHeadingIdx - tocStart, ...tocBlock);
		}
	}

	let finalContent = linesWithAnchors.join('\n');
	if (!finalContent.endsWith('\n')) {
		finalContent += '\n';
	}
	fs.writeFileSync(filePath, finalContent, 'utf8');
}
```

## 🐍 6. Master Python Reference Compiler
<a id="a-6masterpythonreferencecompiler"></a>[TOC](#toc-6masterpythonreferencecompiler)

Below is the production-ready, zero-dependency Python 3 reference implementation of the GFM Markdown standardizer and TOC generator. This corresponds directly to the Python build system used to compile the workspace documentation:

```python
import re
import sys

def process_markdown_file(file_path):
	"""
	Standardizes a single Markdown document in Python:
	1. Generates URL-friendly, lowercase, alphanumeric-only anchor slugs.
	2. Serializes duplicate heading slugs (with unique indexes or timestamp prefixes).
	3. Injects clean raw HTML anchor tags and TOC return-to-top links.
	4. Compiles and updates a nested, beautifully spaced Table of Contents block.
	"""
	try:
		with open(file_path, "r", encoding="utf-8") as f:
			content = f.read()
	except Exception as e:
		print(f"Error reading file {file_path}: {e}", file=sys.stderr)
		return

	# Normalize CRLF to LF
	content = content.replace("\r\n", "\n")
	lines = content.split("\n")

	in_code_block = False
	in_comment_block = False
	headings = []

	# First pass: locate headings
	for i, line in enumerate(lines):
		trimmed = line.strip()

		if trimmed.startswith("```"):
			in_code_block = not in_code_block
			continue
		if in_code_block:
			continue

		# Prevent single-line comment blocks from initiating multi-line comment state
		if not in_comment_block and trimmed.startswith("<!--"):
			if not trimmed.endswith("-->") or trimmed == "<!--":
				in_comment_block = True
			continue
		if in_comment_block:
			if trimmed.endswith("-->") or "-->" in trimmed:
				in_comment_block = False
			continue

		heading_match = re.match(r"^(#{1,6})\s+(.+)$", line)
		if heading_match:
			level = len(heading_match.group(1))
			text = heading_match.group(2).strip()

			if "table of contents" in text.lower():
				continue

			headings.append({
				"line_index": i,
				"level": level,
				"text": text,
				"original_line": line
			})

	if not headings:
		return

	# Assign slugs and handle serialization/duplicates
	current_parent_timestamp = None
	slug_counts = {}

	for heading in headings:
		text = heading["text"]

		# Extract timestamp from parent date headers e.g. ### [2026-07-22T07:11:00Z]
		date_match = re.search(r"\[(\d{4}-\d{2}-\d{2}t\d{2}:\d{2}:\d{2}z)\]", text, re.IGNORECASE) or \
					 re.search(r"\[(\d{4}-\d{2}-\d{2})\]", text)
		if date_match:
			current_parent_timestamp = re.sub(r"[^a-z0-9]", "", date_match.group(1).lower())

		# Generate base slug
		clean_text = re.sub(r"<[^>]+>", "", text)
		base_slug = re.sub(r"[^a-z0-9]", "", clean_text.lower())

		is_log_subheading = any(phrase in text for phrase in [
			"Goals & Requirements", "Primary Goals", "Completed Changes", "Affected Files"
		])

		slug = base_slug
		if is_log_subheading and current_parent_timestamp:
			slug = base_slug + current_parent_timestamp
		else:
			if slug not in slug_counts:
				slug_counts[slug] = 0
			else:
				slug_counts[slug] += 1
				slug = f"{slug}-{slug_counts[slug]}"

		heading["slug"] = slug

	# Second pass: insert/update anchors right below headings
	lines_with_anchors = []
	heading_map = {h["line_index"]: h for h in headings}

	skip_lines_count = 0
	for i, line in enumerate(lines):
		if skip_lines_count > 0:
			skip_lines_count -= 1
			continue

		lines_with_anchors.append(line)

		if i in heading_map:
			h = heading_map[i]
			anchor = f'<a id="a-{h["slug"]}"></a>[TOC](#toc-{h["slug"]})'

			# Check for pre-existing anchor lines to skip and replace
			next_idx = i + 1
			while next_idx < len(lines) and lines[next_idx].strip() == "":
				next_idx += 1
			if next_idx < len(lines):
				next_line_trimmed = lines[next_idx].strip()
				if next_line_trimmed.startswith('<a id="a-') and "[TOC](#toc-" in next_line_trimmed:
					skip_lines_count = next_idx - i

			lines_with_anchors.append(anchor)

	# Third pass: find or insert/replace the Table of Contents block
	toc_start = -1
	for i, line in enumerate(lines_with_anchors):
		if line.strip() == "<!-- TOC location -->":
			toc_start = i
			break

	if toc_start == -1:
		first_h1_idx = -1
		for i, line in enumerate(lines_with_anchors):
			if line.startswith("# ") and "table of contents" not in line.lower():
				first_h1_idx = i
				break

		if first_h1_idx != -1:
			insert_idx = first_h1_idx + 1
			if insert_idx < len(lines_with_anchors) and lines_with_anchors[insert_idx].startswith('<a id="a-'):
				insert_idx += 1
			lines_with_anchors.insert(insert_idx, "")
			lines_with_anchors.insert(insert_idx + 1, "<!-- TOC location -->")
			lines_with_anchors.insert(insert_idx + 2, "---")
			toc_start = insert_idx + 1

	if toc_start != -1:
		next_heading_idx = -1
		for i in range(toc_start + 1, len(lines_with_anchors)):
			line = lines_with_anchors[i]
			trimmed = line.strip()
			if trimmed.startswith("#") and "table of contents" not in trimmed.lower():
				next_heading_idx = i
				break

		if next_heading_idx != -1:
			toc_block = [
				"<!-- TOC location -->",
				"## 🔍 Table of Contents",
				"<!-- Maintained by script -->"
			]

			levels = [h["level"] for h in headings]
			min_level = min(levels) if levels else 2

			for h in headings:
				indent = " " * (max(0, h["level"] - min_level) * 2)
				item_text = re.sub(r"<[^>]+>", "", h["text"]).strip()
				toc_block.append(f"{indent}- [{item_text}](#a-{h['slug']}) <a id=\"toc-{h['slug']}\"></a> ^toc-{h['slug']}")
			toc_block.append("---")

			lines_with_anchors[toc_start:next_heading_idx] = toc_block

	final_content = "\n".join(lines_with_anchors)
	if not final_content.endswith("\n"):
		final_content += "\n"

	try:
		with open(file_path, "w", encoding="utf-8") as f:
			f.write(final_content)
	except Exception as e:
		print(f"Error writing file {file_path}: {e}", file=sys.stderr)
```
